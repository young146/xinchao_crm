# ⚙️ 발행 일정 UI 편집 - 치트시트

## 🚀 빠른 접근

```
대시보드 → 📅 발행 일정 → ⚙️ 일정 편집 모드
```

---

## ➕ 새 호수 추가

```
[호수: 555] [날짜: 2026-03-05] [상태: 예정▼] [➕ 추가]
```

---

## ✏️ 수정하기

### 날짜 수정
```
날짜 클릭 → 수정 → 밖 클릭 (자동 저장)
```

### 상태 변경
```
드롭다운 선택 → 즉시 저장
[예정▼] → [발행됨▼]
```

### 빠른 발행
```
[✓ 발행] 버튼 클릭 → 즉시 "발행됨"
```

---

## 🗑️ 삭제하기

```
[🗑️ 삭제] → 확인 → 삭제됨
```

---

## 💾 백업 & 복원

### 백업
```
[📥 내보내기] → JSON 파일 저장
```

### 복원
```
[📤 가져오기] → JSON 파일 선택 → 확인
```

---

## 🔄 초기화

```
[🔄 초기화] → 확인 → 기본 일정으로 되돌림
```

---

## 📊 UI 상태

```
🔵 현재   (주황색 배경)
✅ 발행됨 (회색 배경)
📘 예정   (파란색 배경)

📆 정상   (14~16일, 녹색)
⚠️ 예외   (18일+, 주황색)
```

---

## 🔄 자동 적용

```
변경 → localStorage 저장 → F5 → 모든 곳 반영
```

---

## ⚡ 실전 예시

### Vol 554 발행
```
1. 목록에서 Vol 554 찾기
2. [✓ 발행] 클릭
3. ✅ 완료!
```

### Vol 575~580 추가
```
반복:
[호수: 575] [날짜: 2026-07-01] [상태: 예정] [➕]
[호수: 576] [날짜: 2026-07-15] [상태: 예정] [➕]
...
```

### 날짜 변경
```
날짜 클릭 → 2026-03-20 선택 → 밖 클릭 → ✅
```

---

## ⚠️ 주의

```
✅ 변경 즉시 저장
⚠️ 브라우저 캐시 삭제 시 데이터 손실
💡 중요 변경 전 백업 (내보내기)
```

---

## 🔧 문제 해결

```
반영 안 됨    → F5 (새로고침)
저장 안 됨    → 콘솔 확인 (F12)
현재 Vol 이상 → status "published" 확인
데이터 사라짐 → 백업 복원 (가져오기)
```

---

## 💡 Best Practice

```
매주: 백업 (내보내기)
변경 전: 확인 & 백업
발행 후: 즉시 status 변경
간격: 14~16일 (격주), 18~21일 (3주)
```
